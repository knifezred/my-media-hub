# UI 设计规范

# My Media Hub

版本：1.0

状态：Draft

---

# 一、设计目标

My Media Hub 的目标不是管理媒体。

而是帮助用户发现媒体。

因此界面设计原则：

```text
发现优先
推荐优先
内容优先
管理最后
```

---

# 二、设计理念

## Discovery First

首页必须立即展示内容。

禁止出现：

* 空白首页
* 功能导航首页
* 配置首页

用户打开应用后应直接看到推荐内容。

---

## Content First

内容是主角。

界面是辅助。

避免：

* 复杂菜单
* 多层导航
* 大量操作按钮

---

## Low Learning Cost

用户无需阅读教程即可使用。

---

## Fast Access

用户点击次数越少越好。

目标：

```text
打开应用
↓
发现内容
↓
进入详情
≤ 3次点击
```

# 二十二、首页成功标准

用户打开应用后：

5秒内看到推荐内容。

30秒内发现感兴趣内容。

无需主动搜索即可开始消费资源。

---

# 二十三、最终目标

My Media Hub 不应该像：

文件管理器。

也不应该像：

数据库后台。

而应该像：

个人版 Netflix

*

个人版 Spotify

*

个人版 Steam

让用户持续发现自己已经拥有却被遗忘的内容。
