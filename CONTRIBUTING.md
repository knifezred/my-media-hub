# Pull Requests

Before submitting a PR:

1. Ensure code compiles.

2. Follow project architecture.

3. Follow API contracts.

4. Follow naming conventions.

5. Update related documentation if required.

---

# Documentation Rules

Do not modify project architecture, database design, API design, or recommendation design without discussion.

Please open an Issue first.

---

# Issues

Bug reports and feature requests are welcome.

When creating an Issue:

Provide:

* description
* expected behavior
* actual behavior
* reproduction steps

if applicable.
